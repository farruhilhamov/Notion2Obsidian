OVERVIEW
--------
Converts Notion databases (including Base list tables) to fully
manageable Obsidian databases using the Dataview plugin.

KEY FEATURES
------------
✅ Automatic database detection from CSV files
✅ Each row → Separate Obsidian note
✅ Properties → YAML frontmatter
✅ Interactive Dataview queries
✅ Multiple view types (table, list, filtered)
✅ Supports all property types (text, number, date, checkbox, etc.)
✅ Preserves data integrity
✅ Easy to manage and extend

HOW IT WORKS
------------
1. Notion exports databases as CSV files
2. Converter detects CSV files automatically
3. Creates folder structure: DatabaseName_Database/
4. Generates individual .md file for each row
5. Creates index file with Dataview queries
6. Install Dataview plugin in Obsidian
7. View and manage database interactively

FILE STRUCTURE
--------------
Input (Notion Export):
  Tasks.csv
    Name, Status, Priority, Due Date, Completed
    Task A, Done, High, 2025-12-20, Yes
    Task B, In Progress, Medium, 2025-12-22, No
    ...

Output (Obsidian):
  Tasks_Database/
    ├── Task A.md          (YAML frontmatter + content)
    ├── Task B.md          (YAML frontmatter + content)
    ├── Task C.md          (YAML frontmatter + content)
    └── Tasks_Index.md     (Dataview queries)

-----------------

DATAVIEW QUERIES
----------------
The index file includes ready-to-use queries:

# Table View
```dataview
TABLE status, priority, due_date
FROM "Tasks_Database"
WHERE contains(tags, "database-item")
SORT due_date ASC
```

# Filtered View
```dataview
TABLE priority, due_date
FROM "Tasks_Database"
WHERE status = "In Progress"
```

# List View
```dataview
LIST
FROM "Tasks_Database"
WHERE completed = false
```

MANAGING YOUR DATABASE
----------------------
Add items:
  - Create new .md file in database folder
  - Add YAML frontmatter with properties
  - Save and it appears in queries

Edit items:
  - Open note file
  - Edit frontmatter values
  - Dataview updates automatically

Delete items:
  - Delete the .md file
  - Removed from all queries

Search/Filter:
  - Use Dataview WHERE clauses
  - Create custom view files
  - JavaScript queries for advanced logic
